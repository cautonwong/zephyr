# 智能电表负荷辨识 IP 组件（Smart Meter NILM IP）产品需求文档（PRD）

---

## 1. 产品概述与商业价值
随着“双碳”目标与“虚拟电厂（VPP）”的快速推进，国家电网与南方电网急需获取居民侧和工商业侧的电器级用能明细，以进行精细化用电负荷预测、需求响应和有序充电管理。
本产品是运行于下一代电网双核智能电表管理模组（基于万高 V32F20x 芯片）的**“非侵入式负荷辨识（NILM）算法 IP”**。它在表端直接解耦家庭总线上的用电负荷类型与功耗，提供无需入户加装采集器的纯软件升级方案。

### 1.1 变现路径与商业模式
*   **软件 IP 授权费（Royalty Model）**：向智能电表制造厂商（威胜、林洋等）按只收取算法授权许可费（如 2~3 元/电表）。
*   **虚拟电厂需求响应数据分成（B2B）**：将脱敏后的电器级用电画像输出给电网调度侧，协助电网完成削峰填谷，按为电网节省的备用容量进行收益分成。
*   **电能表增值检测系统（B2G）**：向电网计量院提供大功率高危用电自动排查与电器老化诊断算法插件。

## 2. 核心功能规格

| 序号 | 功能模块 | 功能描述 | 验收标准 |
|:---|:---|:---|:---|
| 1 | **多稳态负荷识别** | 识别连续运行的稳态电器（冰箱、白炽灯、空调稳态运行）的有功/无功变化。 | 稳态电器功耗识别分辨率误差 $\le 5\text{W}$。 |
| 2 | **暂态特征融合** | 提取电器启动瞬间的暂态浪涌电流峰值、相位角以及 3、5、7 次谐波特征进行多维向量匹配。 | 复杂家电启动识别准确率 $\ge 92\%$。 |
| 3 | **WASM 沙盒热插拔** | NILM 算法与电器特征指纹库作为 `.wasm` 字节码部署，通过 WASM 虚拟机热更新。 | 动态模型升级耗时小于 5 秒，无需电表整机重启。 |
| 4 | **超低功耗常态监听** | 日常维持在极低功耗，仅由 M0 核的功率畸变事件触发 M33 的 tinyML 运算。 | 运算额外增加电表功耗小于 0.05W。 |

## 3. 技术实现方案与外设映射（基于 V32F20x）

### 3.1 沙盒化 WASM 架构设计
智能电表的计量精度和防窃电功能属于电网强制监管范围，任何新功能的引入绝不能污染底层的计量 OS 核心。
*   **WASM 虚拟机集成**：在 M33 核心（[wasm_sandbox.c](file:///workspaces/applications/apps/vango_demo/src/services/wasm_sandbox.c)）中运行 WAMR 解释器，分配独立的 `64KB` 堆内存池。
*   **Host-API 导出设计**：
    *   `host_read_rms_values(float *v, float *i)`: WASM 沙盒动态获取当前的电压和电流有效值。
    *   `host_read_harmonics(int8_t *buffer, size_t size)`: 导出 FFT 计算后的谐波数组。
    *   `host_db_store_event(uint32_t timestamp, char *appliance, float energy)`: 允许沙盒将分类识别结果写入电表的 FlashDB 存储分区。
*   **沙盒隔离性**：沙盒若发生溢出或异常，Zephyr 内核将直接捕获并重启沙盒，保障电表的计量与通信主业务永不瘫痪。

### 3.2 查表化（LUT）前/后处理数学优化
为了应对 M33 核心没有硬件神经网络加速器的现实，大幅降低 tinyML 运算的时钟消耗：
*   **Sigmoid 预计算查找表**：在前向传播中，量化激活函数统一采用有符号量化表查表。
*   **加窗函数与 FFT 因子查找表**：128点 FFT 运算的复数旋转因子 `arm_cfft_sR_f32_len128` 直接由 CMSIS-DSP 库的常数区提供，避免在线计算三角函数。

## 4. 电源管理与并发策略
电表对待机功耗有极其严苛的电网规范要求（通常整表功耗 $< 1.5\text{W}$）。
*   **PM Runtime 联动**：NILM 模块在没有暂态事件时，其对应的虚拟加速器驱动 `sim_axon` 保持在 `PM_DEVICE_STATE_SUSPEND` 状态，控制 GPIO 关闭 M33 部分外设电源。
*   **低优先级调度**：推理任务由低优先级线程 `nilm_wasm_thread` 承载。若电表发生红外/波段串口抄表事件，高优先级的 UART 接收中断（`uart0`）可无时延抢占推理线程，避免抄表超时。

## 5. 验收标准与测试矩阵
1.  **WASM 动态升级测试**：运行 `ota trigger_sim` 命令，验证新的 WASM 负荷辨识模型文件能成功下载到电表 `slot1_ns_partition`，并在不发生整机重启的情况下，热加载新的电器特征库。
2.  **辨识准确率验证**：使用 Python pytest 脚本通过虚拟总线连续发送热水器、微波炉和电视的复合运行电流数据，验证电表控制台能通过 WASM 沙盒正确解析出三者的运行能耗比例。
