# 智能雷达测距与人体存在检测模组（Radar Ranging Module）产品需求文档（PRD）

---

## 1. 产品概述与商业价值
传统的红外热释电（PIR）人体检测传感器在环境温度接近人体（如夏天 35-37℃）时几乎完全失效，且无法检测静止站立的人，极易发生“人在灯灭”的尴尬体验。红外测距和相机识别则容易受烟雾、粉尘和隐私合规限制。
本产品是基于万高 V32F20x 芯片开发的一种**“低成本、超低功耗 24GHz 毫米波雷达/超声波双模融合测距模组”**。它通过高频捕捉雷达芯片输出的中频（IF）模拟信号，在边缘端运行 2D FFT 测距与多目标速度识别算法，广泛应用于智能马桶、人体感应路灯、自动门、机器人防撞以及智能家居。

### 1.1 变现路径与商业模式
*   **高溢价替代传统 PIR 模组（B2B）**：以 30-50 元的价格向智能卫浴、楼宇照明控制面板、智能安防厂商提供毫米波存在传感器电路板（制造成本 $\le 8$ 元），毛利率高达 70%-80%。
*   **工业级 AGV/仓储机器人防撞雷达销售（B2B）**：针对仓储机器人领域提供高精度超声波/雷达测距套件，单套售价 300-500 元。
*   **智能路灯空闲节能增值服务**：向智慧城市集成商提供“车/人流量感应调光系统”软硬件解决方案，按路灯节省的电力电量抽成。

## 2. 核心功能规格

| 序号 | 功能模块 | 功能描述 | 验收标准 |
|:---|:---|:---|:---|
| 1 | **中频雷达信号捕获** | M0 核心对毫米波雷达前端输出的 I/Q 两路中频模拟信号进行连续 ADC 采集。 | 采样率 100kHz，无失真波形重建。 |
| 2 | **一维距离 FFT 提取** | M33 对每个 Chirp（扫频脉冲）的数据运行 128/256 点 Range-FFT，解析出物体的距离峰值。 | 测距精度 $\le 5\text{cm}$，最大探测距离 $\ge 8\text{m}$。 |
| 3 | **二维速度 Doppler FFT** | 结合连续多个 Chirp 的 Range-FFT 结果进行跨 Chirp 的 Doppler-FFT，解析物体是靠近还是远离。 | 支持最小识别速度 $\ge 0.1\text{m/s}$。 |
| 4 | **微安级间歇低功耗工作** | 在不需连续监测的场景（如马桶人体接近），采用 Duty Cycle 间歇式工作，扫频 20ms 后立即休眠 200ms。 | 平均待机电流 $\le 80\text{uA}$。 |

## 3. 技术实现方案与外设映射（基于 V32F20x）

### 3.1 扫频发生器（Chirp）与 SPI 控制
*   **雷达收发器（Tranceiver）控制**：雷达前端收发芯片（如片外 24GHz 传感器）通过 SPI 接口（在设备树中配置为 `spi22`）配置频偏、调频斜率和扫频带宽。
*   **ADC 定时触发采集**：使用 TIM0 定时器产生触发脉冲，使得 ADC 在每次 Chirp 开始的瞬间以 100kHz 速率连续采集 128 点，并将 I/Q 双路数据搬移到 `WAVEFORM_SHM` 中。

### 3.2 2D Range-Doppler 算法在 M33 上的实现
*   **Range-FFT**：
    *   M33 接收到数据后，对每一个 Chirp 数组施加 Hanning 窗，然后调用 `arm_cfft_f32` 得到距离波形。
*   **Doppler-FFT**：
    *   将 16 个 Chirp 的 Range-FFT 结果排列成二维矩阵，沿第二维度（Chirp 轴）再次进行 FFT 变换，提取目标的运动速度。
    *   通过恒虚警率（CFAR）阈值检测器，过滤环境背景热噪声，输出可靠的物体距离与靠近/远离速度特征。

## 4. 验收标准与测试矩阵
1.  **静态测距测试**：在雷达前方 3.0 米处放置一个反射挡板，验证控制台输出 `Target Distance: 3.0m`，误差不超过 $\pm 5\text{cm}$。
2.  **微动存在检测测试**：测试人员在 5.0 米处站立保持静止但有微弱呼吸，验证雷达能持续触发 `Target detected: Presence confirmed (Micro-motion)`，不发生漏报。
